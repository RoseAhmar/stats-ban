# CLAUDE.md — контекст проекта для Claude Code

Этот файл читается автоматически. Здесь — всё, что нужно знать, чтобы продолжить
работу над проектом без предыдущей переписки.

## Что это

Скрипт `update_dashboard.py` тянет задачи из ClickUp, считает статистику и
публикует 4 HTML-страницы на GitHub Pages (`RoseAhmar/stats-ban` →
<https://roseahmar.github.io/stats-ban/>): `index.html`, `dashboard.html`,
`billing.html`, `verif.html`.

## Главная команда

Когда пользователь говорит **«сделай выгрузку и запуш»** — это значит:
запустить `python3 update_dashboard.py` (на Windows был отдельный python; на маке
`python3` из `.venv`). Скрипт сам выгружает данные, генерирует HTML и пушит всё
на GitHub. После запуска — кратко отчитаться цифрами из вывода (задач, закупок,
баеров, верификации).

## Как устроено

- `dashboard_v5_4.html` / `billing_template.html` / `verif_template.html` — шаблоны.
  Скрипт подставляет в них JSON-данные (`const XXX = [...]`) и пишет результат в
  `dashboard.html`+`dashboard_preview.html`, `billing.html`+`billing_preview.html`,
  `verif.html`+`verif_preview.html`.
- Публикация — через GitHub Contents API (`push_file_to_github`), не через
  локальный git. Поэтому на GitHub копятся коммиты «Обновление дашборда …».
  Если понадобится `git push` исходников — сначала `git pull --rebase`.
- Загрузка ClickUp — параллельная (8 воркеров), обычные + архивные задачи,
  `subtasks=true`. У `_fetch_page` есть ретраи (4 попытки, timeout 45с) — при
  таймауте одной страницы выгрузка не обрывается. Если всё же пришло заметно
  меньше задач, чем в прошлый раз — перезапустить.

## Поле «Верификация» (ClickUp dropdown) — ЦЕНТРАЛЬНАЯ ЛОГИКА

Раньше было поле «Проходил вериф» — его **переименовали в «Верификация»**
(`FIELD_VERIF`). Опции: `Нет`, `Да`, `Подан`, `Прошел`, `Не прошел вериф`,
`Прошел+кло`, `Не прошел+кло`, `Кло`.

⚠️ Внимание: опция называется именно **«Не прошел вериф»** (не «Не прошел»).
ClickUp уже один раз её переименовывал — если проценты/теги вдруг «поедут»,
проверь актуальные имена опций в ClickUp и синхронизируй:
- Python: `VERIF_FINAL_STATUSES`, `count_verif`, `verif_success_pct`
- HTML: `VERIF_COLORS`, `VERIF_ORDER` (есть в `dashboard_v5_4.html`, в двух местах).

**Формулы % успеха (одинаково на страницах Баера и Doc Sellers):**
- «Финальные» статусы = `Прошел + Прошел+кло + Не прошел вериф + Не прошел+кло +
  Кло + Да` (без `Подан` — он «в ожидании»).
- Большое число = `Прошел / финальные`.
- Меньшее число «с кло» = `(Прошел + Прошел+кло) / финальные`.

`has_verif = verif_status not in (None, "Нет")`.

## Страницы дашборда (вкладки)

- **Закупки** — по дате *создания* задачи (`date_created`), не выдачи.
- **Баера** и **Doc Sellers** — по «Дате выдачи» (`date_vydachi_ms`) и
  `internal_status == "Выдан"`. Обе переведены на логику поля «Верификация»:
  теги статусов + два процента; у баера снизу ещё секция «Статус аккаунтов»
  (`account_status` — ACTIVE/FREEZE/BAN/… для полноты воронки).
- **Биллинг** — по дате выдачи; под каждой датой показывается разбивка по
  агентствам (`agency_daily`). Замены и VitFarm-free не тарифицируются.

## Константы и настройки в `update_dashboard.py`

- Границы месяцев (00:00 по Брюсселю): `DATE_START_MS` (1 мар), `APR_SPLIT`
  (1 апр), `MAY_1_MS`, `JUN_1_MS`. **При добавлении нового месяца** — завести
  такую же метку и продублировать блок расчёта/вкладку во всех трёх шаблонах
  (кнопка периода, `const XXX_JUN/JUL = …`, дефолтный `activePeriod`, фильтры).
- `MB_TEAM_OVERRIDE` — ручное переопределение MB Team (напр. `Husky MB` →
  `traffic-2`, т.к. в ClickUp стояло неверно).
- `EMAIL_PRICE_BY_ID` — себестоимость почт по `ID_buy`.
- Баер появляется в статистике только когда у аккаунтов заполнены
  `internal_status = "Выдан"` И «Дата выдачи».

## Известные грабли

- После пуша страница может показывать старое из-за кэша — жёсткий сброс
  **Ctrl+Shift+R** (Cmd+Shift+R на маке).
- ClickUp иногда отдаёт таймауты на отдельных страницах — есть ретраи, но при
  явно неполной выгрузке просто перезапусти.
- Токены (`CLICKUP_TOKEN`, `GITHUB_TOKEN`) — в `.env` (в `.gitignore`, в репо не
  попадают). См. `README.md` для установки.
